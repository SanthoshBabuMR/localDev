grunt-contrib-connect

  • port - port to serve asset/app
  • hostname - hostname
  • keepalive - (true) keep the server alive indefinitely
  • livereload - (true) to be used in tandem with grunt-contrib-watch or another task that will trigger a live reload server upon files changing
  • useAvailablePort - look for the next available port after the set port option.
  • base - root directory
  • protocol - string specifying http or https
